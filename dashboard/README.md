# Тендерный штаб

Локальная рабочая сводка для менеджеров по тендерным продажам.

## Запуск

Открывайте проект через локальный HTTP-сервер из корня `ИИ-Агент Тендер`, чтобы интерфейс мог загрузить `dashboard/data/tenders.json`.

## Ежедневное обновление

Агент должен:

1. Взять за основу последний Excel из `reports/tenders`, сохранить активные позиции и добавить новые.
2. Пометить отсутствовавшие в предыдущем отчете позиции как «Новый», сохраненные — как «Повторный».
3. Применить цветовую маркировку приоритетов и решений по правилам `AGENTS.md`.
4. Обновить `dashboard/data/tenders.json` только закупками на территории России, кроме Республики Дагестан.
5. Сохранить неизменными идентификаторы и поля менеджера у существующих активных закупок.
6. Удалить из текущего Excel и JSON завершенные, закрытые, отмененные и просроченные закупки.

Excel-отчеты создаются локально в `reports/tenders`, игнорируются Git и не загружаются в GitHub. Кнопка Excel показывается только когда в `dashboard/data/tenders.json` задан существующий локальный `reportFile`.

Статусы первой версии хранятся в `localStorage` браузера по ключу `prs-tender-manager-status-v1`.
